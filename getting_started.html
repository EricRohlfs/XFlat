Getting Started

Maven

 If using Maven, add the following to your POM: 
<dependency>
  <groupId>org.xflatdb</groupId>
  <artifactId>xflat</artifactId>
  <version>0.9</version>
  <classifier>sources</classifier>
</dependency>
Otherwise, you will need the following JARs to use XFlat:
  • jdom-2.0.4.jar or newer
  • hamcrest-core-1.3.jar or newer
  • hamcrest-library-1.3.jar or newer
  • commons-logging-1.1.1.jar or newer

If you want to use queries involving XPath expressions, you will need Jaxen.

  • jaxen-1.1.4.jar or newer
Note that while you do not strictly need Jaxen to run XFlat, you will not be able to use the following features without it:
  • Any operations using XPathQuery
  • Any operations using XPathUpdate
  • Sharded Tables

If you want to take advantage of automatic POJO mapping using JAXB, you will need the JAXB Reference Implementation jars. You do not need JAXB if you register your own conversions with the database's conversion service.

  • jaxb-api.jar
  • jaxb-impl.jar
  • jaxb-xjc.jar

Setup

Creating an XFlat instance with the default configuration is incredibly easy, simply point it to the directory where you want it to store your data. No configuration is necessary if you are okay with the default options. {% highlight java %} Database myDatabase = XFlatDatabase.Build(new File("my/data/directory")).create(); {% endhighlight %} If you need additional configuration options, set the database configuration before calling initialize. {% highlight java %} DatabaseConfig dbConfig = DatabaseConfig.DEFAULT .withThreadCount(2); Database myDatabase = XFlatDatabase.Build(directory) .withDatabaseConfig(dbConfig) .create(); {% endhighlight %} If you need to configure any tables, do that here as well. Tables do not need to be configured in order to be used; they only need to be configured if you want to change the default settings. {% highlight java %} Database myDatabase = XFlatDatabase.Build(directory) .withTableConfig("foo", TableConfig.DEFAULT .withIdGenerator(MyCustomIdGenerator.class)) .create(); {% endhighlight %} If you have a custom serializer and deserializer, you can register that on the conversion service. Do this before you use the database. {% highlight java %} Database myDatabase = XFlatDatabase.Build(directory).create(); db.getConversionService().addConverter(Foo.class, Element.class, new FooToElementConverter()); db.getConversionService().addConverter(Element.class, Foo.class, new ElementToFooConverter()); {% endhighlight %} The database configuration options are enumerated in the DatabaseConfig class.
The table configuration options are enumerated in the TableConfig class.
